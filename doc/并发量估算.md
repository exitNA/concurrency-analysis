# AI智能体（对话类）并发量估算技术文档

## 1. 适用场景

适用于对话类AI智能体，仅已知日总对话数、无需历史数据/时段分布，快速估算并发量，用于接口设计、限流配置及模型部署选型。

## 2. 核心指标定义

- D：日对话次数（每日总对话轮次，唯一核心已知指标）

- T：单轮对话平均耗时（秒，按行业通用值取值，无需实测）

## 3. 并发量估算公式（基础版）

### 3.1 平均并发

#### 建模原理

对话类AI的流量存在明显日间/深夜分布差异（深夜流量近零），符合“核心负载集中于日间”的场景假设——系统负载主要由日间对话请求产生，深夜低流量对整体负载影响可忽略。通过切分日间流量、聚焦日间活跃时长，量化日间平均负载，避免全天平均导致的并发低估，为系统基础负载评估提供支撑。

定义：日间平均每秒同时等待回复的用户数（核心参考，规避全天平均导致的低估）。

核心修正：流量存在明显日间/深夜划分（深夜流量近零），按“日间流量占比”切分计算，忽略深夜流量影响。

核心公式： $并发_{avg(日间)} = \frac{D \times 日间流量占比 \times T}{日间活跃时长(秒)}$ 

参数说明（通用取值，无需实测）：
      1.  日间流量占比：90%~95%（对应深夜5%~10%）；
      2.  日间活跃时长：普通AI 8~10小时（28800~36000秒），电商/活动类6~8小时（21600~28800秒）。

补充（无昼夜差异场景）： $并发_{avg} = \frac{D \times T}{86400}$ 

### 3.2 峰值并发

定义：日间高峰时段最大并发量（系统需承载的极限负载）。

核心公式： $并发_{peak} = 并发_{avg(日间)} \times 峰值系数$ 

## 4. 进阶统计模型（仅需日总对话数）

无需复杂建模、无需历史数据，精准估算高峰并发及限流阈值。

### 4.1 简化时间序列峰值模型

#### 建模原理

对话类AI的日间流量存在明显高峰时段，符合“高峰时段对话量集中”的时间序列分布假设——日间某1小时内的对话量占日总对话数的比例，符合行业通用规律。通过日总对话数结合高峰占比，精准估算高峰时段瞬时对话量，捕捉系统极限负载，为高峰限流、资源扩容提供精准参考。

核心逻辑：用日总对话数×高峰占比，估算高峰1小时对话量，精准捕捉瞬时负载。

核心公式： $并发_{peak} = \frac{D \times 高峰占比 \times T}{3600}$ 

参数说明：高峰占比按应用类型取值（无需实测），详见第5节参数表。

### 4.2 泊松分布统计模型（简化版）

#### 建模原理

对话类AI的用户请求发起具有「随机性」（用户发起对话无固定时间规律，且单次对话请求相互独立），符合泊松分布的核心假设——单位时间内（1秒），随机事件（对话请求）发生的次数服从泊松分布。通过泊松分布可量化“某一秒内并发量达到某一阈值”的概率，为高可用设计（如限流、容灾）提供数据支撑，避免仅凭经验设定阈值导致的风险。

#### 核心模型公式

泊松分布概率公式（核心，用于估算并发量≥k的概率）： $P(X \geq k) = 1 - \sum_{i=0}^{k-1} \frac{e^{-\lambda} \lambda^i}{i!}$ 

参数说明：
1.   $\lambda$ ：每秒平均请求数（同日间平均并发，无需额外计算）；
2. $k$ ：预设的并发阈值；
3.   $e$ ：自然常数（约2.718）；
4.   $i!$ ：i的阶乘（如3! = 3×2×1）。

#### 具体示例（贴合实际场景，步骤可直接套用）

基于前文示例已知条件：D=10万次、T=2秒、日间流量占比=90%、日间活跃时长=36000秒（10小时），结合泊松分布核心公式，完整演算限流阈值确定过程，无需复杂手动求和，兼顾精准性与实用性。

步骤1：计算每秒平均请求数λ（同日间平均并发）

代入公式： $\lambda = \frac{D \times 日间流量占比 \times T}{日间活跃时长(秒)}$ 

计算过程： $\lambda = \frac{100000 \times 90\% \times 2}{36000} = \frac{180000}{36000} = 5.0$ 

步骤2：明确泊松分布核心公式应用目标

需估算“并发量≥k”的概率，核心公式： $P(X \geq k) = 1 - \sum_{i=0}^{k-1} \frac{e^{-\lambda} \lambda^i}{i!}$ ，其中λ=5.0，e≈2.718。

步骤3：分场景演算k值（并发阈值）及对应超标概率

场景1：测试环境（允许较高超标概率，临时阈值）——取k=5

计算过程： $P(X \geq 5) = 1 - \left( \frac{e^{-5} \times 5^0}{0!} + \frac{e^{-5} \times 5^1}{1!} + \frac{e^{-5} \times 5^2}{2!} + \frac{e^{-5} \times 5^3}{3!} + \frac{e^{-5} \times 5^4}{4!} \right)$ 

简化计算（无需手动求和）：代入e≈2.718，计算得P(X≥5)≈5.3%，符合测试环境临时阈值需求。

场景2：常规生产环境（99.5%稳定性，控制超标概率≤0.5%）——取k=7

简化计算：基于λ=5.0，代入公式可快速得P(X≥7)≈0.5%，满足常规生产环境稳定性要求。

场景3：企业级应用（99.9%稳定性，控制超标概率≤0.1%）——取k=8

简化计算：基于λ=5.0，代入公式可快速得P(X≥8)≈0.1%，适合对稳定性要求高的企业级部署，推荐设为限流阈值。

步骤4：示例结论

当λ=5.0时，可根据稳定性需求选择限流阈值：测试环境设为5，常规生产环境设为7，企业级应用设为8~10，与前文简化表格完全匹配，验证了公式的实用性。

|k值（并发阈值）|P(X≥k)（超标概率）|应用建议|
|---|---|---|
|5|≈5.3%|测试环境，临时阈值|
|7|≈0.5%|常规生产环境（99.5%稳定性）|
|8|≈0.1%|企业级应用（99.9%稳定性）|
简化技巧：k值≈λ+3~λ+5，日间流量占比可直接取90%。

## 6. 示例演算

已知：D=10万次，应用为“有明显高峰”，取通用参数：T=2秒、峰值系数=8、高峰占比=30%、日间流量占比=90%、日间活跃时长36000秒（10小时）

1. 日间平均并发：(100000×90%×2)÷36000≈5.0；

2. 基础版峰值并发：5.0×8≈40~45；

3. 时间序列模型峰值并发：(100000×30%×2)÷3600≈16.7；

4. 泊松分布限流阈值：λ=5.0，需99.9%稳定性，设为10~12。

## 7. 核心结论

1.  仅需日总对话数即可估算并发，日间/深夜流量差异显著时，按90%~95%日间占比计算，避免低估；

2.  核心依赖日总对话数+行业通用参数，无需实测、无需历史数据；

3.  系统部署优先参考峰值并发，单轮对话耗时越长，并发压力越大。
